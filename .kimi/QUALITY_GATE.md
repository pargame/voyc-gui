# 코드 품질 검수 게이트

## 1. 하드코딩 검사

- [ ] 모든 숫자 리터럴이 `Config.qml`에 정의되어 있는가
- [ ] 모든 색상이 `Config.qml` 테마 함수를 통해 분기되는가
- [ ] 여백/패딩/폰트 크기가 `Config.qml`에 있는가
- [ ] 비율/비율값이 `Config.qml`에 있는가

## 2. 통합 UI 검사

- [ ] 버튼/항목 등 재사용 가능한 UI 요소가 `UiButton` 등 컴포넌트로 분리되어 있는가
- [ ] `hovered` 상태가 외부에서 주입되는가
- [ ] 이벤트 처리가 시각적 요소와 분리되어 있는가

## 3. 이벤트 시스템 검사

- [ ] `MouseArea`가 중복되거나 겹치지 않는가
- [ ] `z` 순서가 명확하게 설계되어 있는가
- [ ] 팝업/오버레이가 `Window` 레벨 또는 별도 `Layer`에 있는가
- [ ] 동적 생성된 객체의 참조(`id`)가 부모 변경 후에도 유효한가

## 4. 테마 시스템 검사

- [ ] 다크/화이트 모드 색상이 쌍으로 정의되어 있는가
- [ ] 새로운 색상 추가 시 `PROGRESS.md` 색상 테이블에 기록되는가

## 5. 진도 문서 검사

- [ ] 기능 완성 후 `PROGRESS.md`에 기록되었는가
- [ ] 코드 구조 변경 시 Config 분류 체계도 갱신되었는가

## 6. 커밋 전 검사

- [ ] `.gitignore` 점검
- [ ] `git status`로 변경 파일 확인
- [ ] 레거시 코드(미사용 함수, 주석 처리)가 있는가
