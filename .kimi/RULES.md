# voyc-gui 코딩 규칙

## 1. 하드코딩 금지

- 모든 숫자, 색상, 크기, 비율은 `Config.qml`에 정의
- `Config` 참조 ID는 `cfg`로 통일
- 예외: 문자열 리터럴(제목, 메뉴 항목), 수학 공식(`radius: width / 2`)

## 2. 논리적 구조

- 앵커 기반 레이아웃 우선, 좌표 하드코딩 지양
- 좌표 변환 함수(`toWorld`, `toScreen`) 등 재사용 가능하게 분리
- 비슷한 기능은 함수/컴포넌트로 추상화

## 3. 테마 시스템

- 색상은 `Config.qml` 함수로 분기 (`isDarkMode`)
- 새로운 색상 추가 시 다크/화이트 모드 둘 다 정의
- 색상 분류는 PROGRESS.md 색상 테이블에 기록

## 4. 진도 관리

- 기능 완성 후 즉시 `src/PROGRESS.md` 업데이트
- 완료 항목: 기능명, 설명, 날짜
- 코드 구조 변경 시 Config 분류 체계도 함께 갱신

## 5. 품질 검사

- 주기적으로 전체 코드 하드코딩 여부 점검
- `grep -n "[0-9]\+" src/*.qml` 등으로 숫자 리터럴 확인
- 레거시 코드(미사용 함수, 주석 처리된 코드) 발견 시 사용자에게 삭제 여부 확인

## 6. 레거시 처리

- 삭제 전 반드시 사용자에게 보고
- `.gitignore`, `CMakeLists.txt` 변경 시 이유 명시
- 문서(`README`, `PROGRESS`, `CHANGELOG`)와 코드 동기화 유지

## 7. 커밋 규칙

- 커밋 전 `.gitignore` 점검
- 푸시 전 `git status`로 변경 파일 확인
- 커밋 메시지는 동사 원형으로 시작 (`Add`, `Fix`, `Refactor`)
